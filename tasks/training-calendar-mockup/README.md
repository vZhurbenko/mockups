# Мок-ап: Календарь тренировок

## 📋 Описание

Базовый мок-ап календаря для управления тренировками и играми. Демонстрирует **принцип построения** UI-компонентов для календаря событий.

> ⚠️ **Важно:** Это мок-ап для демонстрации архитектуры и UX-паттернов. Дизайн (цвета, оформление, состав карточек) может отличаться в финальном приложении.

---

## 📦 Компоненты

### 1. **Календарь (CalendarGrid)**

**Назначение:** Отображение сетки месяца с событиями.

**Структура:**
- Заголовок с навигацией (пред. месяц / сегодня / след. месяц)
- Дни недели (Пн, Вт, ..., Вс)
- Ячейки дней (6 рядов × 7 колонок)

**Состояния ячейки:**
- Обычный день
- Текущий день (выделен фоном)
- Выбранный день (рамка)
- Выходные (Сб, Вс — красный текст)
- С событиями (краткое описание на desktop, точки на mobile)

**Адаптивность:**
- Desktop: краткое описание событий в ячейке
- Mobile: точки-индикаторы + панель событий снизу

---

### 2. **События месяца (EventsPanel)**

**Назначение:** Список всех событий месяца с группировкой по дням.

**Структура:**
- Фильтр: "Все события" / "Предстоящие"
- Группы по дням (заголовок + карточки событий)
- Разделитель между событиями в одном дне

**Карточка события:**
```
┌───────────────────────────────────────────┐
│  Название события              [Тип]      │
│  Соперник: Eagles BC         [Записаться]│
│                                           │
│  🕐 14:00 - 16:00                         │
│  📍 Стадион Динамо, пр. Ленина, 45       │
│  👥 5 / 10 участников (свободно: 5)      │
│                                           │
│  [Разделитель]                            │
│                                           │
│  👥 Участники (раскрывается):             │
│     — Иванов Алексей                      │
│     — Петров Дмитрий                      │
└───────────────────────────────────────────┘
```

**Состояния кнопки:**
- "Записаться" — есть места
- "Отписаться" — пользователь записан
- "В резерве" — пользователь в waitlist
- "Завершено" — дата прошла

---

### 3. **Модалка дня (DayModal)**

**Назначение:** Просмотр и управление событиями выбранного дня.

**Триггер:** Клик на день в календаре.

**Структура:**
```
┌───────────────────────────────────────────┐
│  Понедельник, 5 мая           [+ Добавить]│
│  2 события                     [✕]        │
├───────────────────────────────────────────┤
│                                           │
│  ┌─────────────────────────────────────┐ │
│  │  Тренировка по баскетболу  [Трен.]  │ │
│  │  🕐 19:00 - 21:00                   │ │
│  │  📍 СК Олимпийский                  │ │
│  │  👥 3 / 12 (свободно: 9)   [Записаться]│
│  └─────────────────────────────────────┘ │
│                                           │
│  ┌─────────────────────────────────────┐ │
│  │  Тренировка по футболу     [Трен.]  │ │
│  │  🕐 17:00 - 18:30                   │ │
│  │  📍 Поле №3                         │ │
│  │  👥 1 / 16 (свободно: 15) [Записаться]│
│  └─────────────────────────────────────┘ │
└───────────────────────────────────────────┘
```

**Особенности:**
- Кнопка "Добавить событие" (для будущих дат)
- Список всех событий дня
- Быстрая запись/отписка

---

### 4. **Модалка добавления события (AddEventModal)**

**Назначение:** Создание нового события.

**Поля формы:**
- Тип события: Тренировка / Игра (radio)
- Название * (text)
- Время начала * (time)
- Время конца * (time)
- Место * (text)
- Адрес * (text)
- Макс. участников (number, 1-50)
- Описание (textarea)

**Валидация:** Обязательные поля помечены `*`

## 🎨 UX-паттерны

### 1. **Двухколоночная карточка события**

```
┌─────────────────────┬─────────────┐
│  Информация         │  Действия   │
│  - Название         │  - Бейдж    │
│  - Время           │  - Кнопка   │
│  - Место           │             │
│  - Участники       │             │
└─────────────────────┴─────────────┘
```

**Преимущества:**
- Компактное размещение
- Кнопка всегда на виду
- Адаптивность (на mobile колонки становятся в ряд)

---

### 2. **Waitlist (резервный список)**

**Логика:**
1. Пользователь записывается
2. Если места есть → основной участник
3. Если мест нет → резерв (waitlist)
4. При отписке основного → первый из резерва становится основным

**Отображение:**
```
Участники (3 / 10):
┌─────────────────────┐
│ Основные (2):       │
│ — Иванов Алексей    │
│ — Петров Дмитрий    │
├─────────────────────┤
│ В резерве (1):      │
│ — Сидоров Сергей    │
└─────────────────────┘
```

---

### 3. **Блокировка прошедших событий**

**Логика:**
- Если дата < текущей → кнопка "Завершено"
- Нельзя записаться/отписаться
- Событие показывается серым

---

### 4. **Фильтр событий**

**Стиль:** Сегментированный контроль (как в ActionButtons.vue)

```
┌─────────────────────────────────────┐
│ Показать: [Все события] [Предстоящие]│
└─────────────────────────────────────┘
```

**Состояния кнопок:**
- Активная: `bg-primary-50 border-primary-200 text-primary-600`
- Неактивная: `bg-white border-gray-300 text-gray-600`

---

### 5. **Табы навигации**

**Стиль:** Material Design с белым фоном активного таба

```
┌─────────────────────────────────────┐
│ [› Календарь] | [События месяца]    │
│ ────────────                        │
└─────────────────────────────────────┘
```

**Элементы:**
- Стрелочка `›` слева от активного таба
- Подчёркивание активного таба
- Разделитель между табами
- Белый фон активного (контраст с серым фоном)

## 🔧 Техническая реализация

### Стек
- **Vue 3** (Composition API, `<script setup>`)
- **Pinia** (state management)
- **Tailwind CSS 4** (стилизация)
- **Lucide Vue Next** (иконки)

### Структура файлов
```
src/
├── views/
│   └── TrainingCalendar.vue       # Основная вьюха
├── components/
│   └── training-calendar/
│       ├── CalendarHeader.vue     # Заголовок с навигацией
│       ├── CalendarGrid.vue       # Сетка календаря
│       ├── CalendarDay.vue        # Ячейка дня
│       ├── EventsPanel.vue        # Список событий
│       ├── EventCard.vue          # Карточка события
│       ├── EventParticipants.vue  # Участники события
│       ├── DayModal.vue           # Модалка дня
│       └── AddEventModal.vue      # Модалка добавления
└── stores/
    └── trainingCalendar.js        # Pinia store
```

### Store (trainingCalendar.js)

**State:**
- `currentDate` — текущий отображаемый месяц
- `selectedDate` — выбранный день
- `events` — массив событий
- `currentUserId` — ID текущего пользователя

**Getters:**
- `currentYear`, `currentMonth`
- `selectedDateEvents` — события выбранного дня
- `upcomingEvents` — предстоящие события
- `monthEvents` — события текущего месяца

**Actions:**
- `goToPreviousMonth()`, `goToNextMonth()`, `goToToday()`
- `selectDate(date)`
- `toggleParticipation(eventId)` — запись/отписка
- `isParticipating(eventId)` — проверка участия
- `isOnWaitlist(eventId)` — проверка waitlist
- `getWaitlistCount(eventId)` — количество в waitlist

## ⚠️ Ограничения мок-апа

1. **Дизайн:** Цвета и оформление могут отличаться в финальном приложении
2. **Данные:** Моковые данные, нет реального API
3. **Авторизация:** Нет разделения прав доступа
4. **Валидация:** Базовая, нет сложной валидации форм
5. **Производительность:** Нет оптимизации для большого количества событий
